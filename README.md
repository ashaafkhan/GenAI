# GenAI Prompting Playground

This repository is a small Node.js playground for experimenting with prompt styles, API clients, weather tooling, and shell/file generation flows.

It started as a minimal OpenAI demo and then grew into a set of prompt examples under `prompting/` that show how different prompting patterns behave in practice.

## What Is In This Repo

- `hello.js` is the simplest OpenAI example.
- `token.js` demonstrates tokenization with `tiktoken`.
- `prompting/01_zero.js` shows zero-shot prompting.
- `prompting/02_few_shot.js` shows few-shot prompting.
- `prompting/03_cot.js` shows chain-of-thought style prompting.
- `prompting/04_cot_tools.js` shows tool use, weather lookup, and command execution.
- `.env.example` documents the environment variables the repo expects: `OPENAI_API_KEY`, `GROQ_API_KEY`, and `ANTHROPIC_API_KEY`.



## How The Prompting Demos Differ

### Zero-shot

The model gets a direct instruction with no examples. This is the simplest style, but it can be less stable.

### Few-shot

The prompt contains examples of the expected answer format. This usually improves consistency.

### Chain-of-thought

The prompt asks the model to reason in steps before answering. In this repo, the pattern is explored as a learning exercise.

### Tools + Reasoning

`04_cot_tools.js` introduces a small agent loop:

- The model returns a JSON step such as `INTIAL`, `THINK`, `TOOL_REQUEST`, `ANALYSE`, or `OUTPUT`.
- If the step is `TOOL_REQUEST`, the script calls the requested tool.
- The tool result is fed back into the conversation.
- The loop continues until the model returns `OUTPUT`.

This file also has defensive parsing logic because some models sometimes return multiple JSON objects or noisy output in a single response.

## Files Generated By The Tool Demo

`prompting/04_cot_tools.js` may generate output such as:

- `prompting/weather.txt`
- `prompting/weather/`

These are runtime artifacts, not source files.

## Notes About The Tool Demo

- The weather tool uses `wttr.in` to fetch current conditions.
- The command-writing examples are written for `cmd.exe` style commands.
- Because the model is asked to produce JSON, the script expects valid JSON output and includes fallback parsing to reduce crashes.

## Current Package Dependencies

- `openai` for chat completion calls.
- `dotenv` for loading environment variables.
- `axios` for weather HTTP requests.
- `tiktoken` for tokenization examples.

## Suggested Usage Pattern

If you are just learning the repo, start in this order:

1. Read `hello.js`.
2. Read `token.js`.
3. Run `01_zero.js`, `02_few_shot.js`, and `03_cot.js`.
4. Study `04_cot_tools.js` last, because it combines prompt design, tool calling, JSON parsing, and Windows command execution.

## Troubleshooting

- If you get a missing credentials error, check that `.env` exists and contains the right key name.
- If a script cannot find `.env`, make sure you are running it from the folder layout it expects.
- If `04_cot_tools.js` crashes on JSON parsing, the model likely returned malformed JSON; the script now has a safer parser, but the prompt still needs to be strict.
- If weather output looks repeated, verify the URL interpolation inside the weather helper.

## Security

- Never commit `.env`.
- Keep real API keys out of screenshots, logs, and prompt files.
- Use `.env.example` for documentation and onboarding.
